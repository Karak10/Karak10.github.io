crossorigin attribute

The crossorigin attribute, used on the <audio>, <img>, <link>, <script> and <video> elements, provides support for CORS (Cross-Origin Resource Sharing).

Cross-Origin Resource sharing aka CORS is an HTTP-heder based mechanism that allows a server to indicate any other origins (domin, scheme, or port) than its own from which a browser should permit loading of resources. CORS also relies on mechanism by which browsers make a "preflight" request to the server hosting the cross-origin source, in order to check that the server will permit the actual request. In tht preflight, the browser sends headers that indicate the HTTP method and headers tht will be used in the actual request.

The cross-origin attribute lets you decide wether you want to make the cross-origin request to get a source from another domain server anonymoulsy or send information while making the request, to send informations use the use-credentials value, else use anonymous value to make the request anonymously.
