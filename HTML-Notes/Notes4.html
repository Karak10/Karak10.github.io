Content Models:

In HTML, elements typically belonged in either the block level (<p>,<h1>,<h2>,<h3>,<ul>,<ol>,<table>,<img>,etc) or inline content model (<strong>,<em>,<a>,<i>,etc). HTML5 introduces seven main content models.

  1. Metadata
  2. Embedded
  3. Interactive
  4. Heading
  5. Phrasing
  6. Flow
  7. Sectioning

Here are more about each model:

Metadata:
Content that sets up the presentation or behavior of the rest of the content. These elements are found in the <head> section of the document.
Elements: <base>,<link>,<meta>,<noscript>,<script>,<style>,<title>
Embedded:
Content that imports other sources into the document.
Elements: <audio>,<video>,<canvas>,<iframe>,<img>,<math>,<object>,<svg>
Interactive:
Content specifically intended for user interaction.
Elements: <a>,<audio>,<video>,<button>,<details>,<embed>,<iframe>,<img>,<input>,<label>,<object>,<select>,<textarea>
Heading:
Defines a section header.
Elements: <h1>,<h2>,<h3>,<h4>,<h5>,<h6>
Phrasing:
This model has a number of inline elements in common with HTML4.
Elements: <img>,<span>,<strong>,<label>,<br>,<small>,<sub>, and more.

The two last models are the following:

Flow content:
Contains the majority of HTML5 elements that would be included in the normal flow of the document.
Sectioning content:
Defines the scope of the headings, content, navigation, and footers.
Elements: <article>,<aside>,<nav>,<section>,<footer>,<header>

About audio element:

There are two ways to put audio in your page, the first one is to simply type <audio src="name of audio" controls> Example </audio>, the text you write between the <audio> and </audio> tags will be displayed if the audio is not supported by a browser, can't be more simple

The second way is a little more complicated but allows you to put more than one audio types so if one isn't supported the other plays, for example, you can put one in mp3, one in ogg, etc, there are many types, and not all browsers support mp3 or ogg, to do this you write <audio controls> <source src = "audio name" type="audio/type"> Audio not supported in your browser </audio>, the type for mp3 is audio/mpeg, for ogg its audio/ogg, if you want to put more audio types you just make another <source> tag, after you have added all the audio types you want you should write the message you want it to have if the audio isn't supported by a browser, the browsers will automatically pick to play the first available type of audio they find, however, there is always a chance one browser may not support any of the types you added, so having this message is good, the controls attribute is used to add the pause and play controls, to change the position of where the audio will be, we can place it inside a praragraph, or use CSS.

Code:

<p align="center">
<audio controls>
<source src="../images/Castelvania.mp3" type="audio/mpeg">
<source src="Castelvania.ogg" type="audio/ogg">
Audio not supported by your browser
</audio>
</p>

Output:

Output:

Another element you can use between the <video> and </video> tags is <track> element, used to specify subtitles, caption files or files containing text, that should be visible when the media is playing. Tracks are formatted in WebVTT format (.vtt files).

Some attributes the track element uses are the following: default specifies that the track is to be enabled if the user's preferences do not indicate that another track would be more appropriate. Kind specifies the kind of the text track, for example, captions, chapters, descriptions, metadata or subtitles. Label specifies the title of the track, src is required, specifies the URL of the track file and lastly, srclang specifies the language of the track text data, (required if kind="subtitles").

The track element can be used for both video and audio elements.

HTML5 Web Storage API:

This is a quick description of how storage works, don't care too much about it for now, you will understand it better in the future when you learn JavaScript.

With HTML5 web storage, websites can store data on a user's local computer.
Before HTML5, we had to use JavaScript cookies to achieve this functionality.

The Advantages of Web Storage

  • More secure
  • Faster
  • Stores a larger amount of data
  • Stored data is not sent with server request

    • There are two types of web storage objects:

  • SessionStorage()
  • LoacalStorage()

    • Local vs Session

  • Session Storage is destroyed once the user closes the browser
  • Local storage stores data with no expiration date

    • The syntax for web storage for both local and session storage is very simple and similar. The data is stored as a key/value pairs.

    Storing a Value:
    LocalStorage.setItem("key1","value1");

    Getting a value:
    (this will print the value)
    alert(localStorage.getItem("key1:));

    Removing a value
    localStorage.removeItem("key1");

    Removing all values:
    localStorage.clear();

    These scripts are written in JavaScript language, to use JavaScript you have to write type <script> your script </script>, thats all I have for now, its not important to understand how storage works at the moment, but its good to know some stuff.

    What is the Geolocation API?

    In HTML5, the Geolocation API is used to obtain the user's geographical location.

    Since this can compromise user privacy, the option is not available unless the user approves it.

    The Gelocation API's main method is getCurrentPosition, which retrieves the current geographic location of the user's device.

    navigator.geolocation.getCurrentPosition();

    Parameters:
    shownLocation (mandatory): Defines the callback method that retrieves location information.
    ErrorHandler (optional): Defines the callback method that is invoked when an error occurs in processing the asynchronous call.
    Options (optional): Defines a set of options for retrieving the location in information.

    Presenting Data

    User location can be presented in two ways:
    Geodetic and Civic

    1. The geodetic way to describe position refers directly to latitude and longitude.
    2. The civic represantation of location data is presented in a format that is more easily read and understood by the average person.

    Each parameter has both a geodetic and civic representation:

    Attribute Geodetic Civic
    Position 59.3, 18.6 Stockholm
    Elevation 10 meters 4th floor
    Heading 234 degrees City center
    Speed 5km/h Walking
    Orientation 35 degrees North-East

    All attributes and input types I know

    Here are all the attributes I know:

    autofocus
    Used in an inpute or textarea, allows the user to input data immediately after the screen has loaded.
    action
    Used in form element, this decides what page opens after you sumbit.
    method
    Used in form element, it can be method="get" or method="post", when method is get the form data is visible in the page address, when its post the form is updating data, or includes sensitive information (passwords).
    enctype
    Used in form element, specifies how the form-data should be encoded when submitting it to the server, this is only used if method="post" and you don't need to know much about it for now.
    target
    Used in form element, you can write target="_blank" so the page opens in another tab.
    validate
    Used in form element, don't know what it dows
    height and width
    Used in input elements to change the height and width, is only used for type="image", for other inputs you can change the size with CSS, or for textarea you can use rows and cols.
    list
    Its used in input element when you want to add a datalist.
    min and max
    Used in input elements, its used to set a min number and a max number for input types like numbers.
    placeholder
    Used in input elements, puts a description of what you have to type
    required
    makes the input required to be filled
    multiple
    Its used in inputs, I haven't talked about it because it wasn't needed, this allows the user to enter more than one values, its however used only for the following input types: email and file. To send multiple files you hold ctrl and shift while selecting, to add multiple emails you put a coma after each email
    step
    Used in input elements, didn't talk about it before, the step attribute specifies the interval between legal numbers in an input element, if step="3" then the legal numbers could be -6, -3, 0, 3, 6, etc, the step attribute can be used together with the max and min attributes to create a range of legal values, step attribute works with the following input types: number, range, date, datetime-local, month, time and week.
    pattern
    Used in input elements, specifies a regular expression that the input element's value is checked against on form submission, for example, to make an input that can contain only three letters with no numbers or special characters you write pattern="[A-Za-z] {3}"
    Title
    Used in input elements, can be used in other elements as well like paragraphs or images, if you write title="something" when you point your mouse on the input a description writing "something" will appear
    disabled
    Used to disable an input from being used
    minlength/maxlength
    Used to restrict the number of characters that can be used in an input
    formaction
    Specifies where to send the form-data when a form is submitted, this attribute overrides the form's action attribute
