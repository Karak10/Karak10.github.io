The CSS Box Model

All HTML elements can be considered as boxes. The CSS box model represents the design and layout of the site. It consists of margins, borders, paddings and the actual content.

The properties work in the same orderL top, right, bottom and left.

The image below illustrates the box model.

When working with boxes, it is important to understand how the total width of an element is calculated. For example, the total width for the box with padding will be the sum of width plus padding lef and padding right.

The total height is calculated the same way as the width. The example below is the same box from the previous picture with padding, border and margin.

You can use padding to define the space between the content and the border, note that any background-color you assign to the content will also be applied to the padding, you can use margin to change the distances of the content in relation to the whole space of the page it's located in, and border of course to add a border that seperates the content/padding from the margin.

The border Property

The border property allows you to customize the borders of HTML elements. In order to add a border to the element, you need to specify the size, style and color of the border. The example below shows how to ad a 5px solid green border to a paragraph.

HTML code:

<p id="border-example"> I'm a paragraph and I have a border around me </p>

CSS code:

<Style> #border-example { border: 5px solid green; width: 300px; margin-left:auto; margin-right:auto; padding:20px; } </Style>

Outcome:

I'm a paragraph and I have a border around me

The properties for the border can be set seperately. The border-width property specifies the width of the border, the border-color specifies the color of the border and the border-style specifies the style of the border, note that if you don't specify a style none of the other border properties will not work.

HTML code:

<p id="border-example2"> I'm a paragraph and I have a border around me </p>

CSS code:

<Style> #border-example2 { border-width: 5px; border-style: solid; border-color: green; width: 300px; margin-left:auto; margin-right:auto; padding:20px; } </Style>

Outcome:

I'm a paragraph and I have a border around me

The default value for border-style is none, which defines no border. There are various styles supported for the border-style property: dotted, dashed, double, etc.

Width and Height

To style an element to have a total width and height you can use the width and height properties.

Code:

<div style=" width: 200px ; height: 200px ; background-color: blue; margin: auto;"> </div>

Outcome:

The width and height of an element can also be assigned using percents

Code:

<div style=" width: 50%; height: 200px; background-color: blue; margin: auto;"> </div>

Outcome:

Using percents instead of length can be prefered in some cases, if you try to change the size of the browser for example the size of the div will also change, so when we access the page on phone for example the div will get 50% width of the phone screen.

There are four more width and height properties we can use:

min-width - the minimum width of an element

max-width - the maximum width of an element

min-height - the minimum height of an element

max-height - the maximum height of an element

These properties can be useful and can help you build responsive websites.

The background-image Property

The background-image property sets one or several background images in an element. Let's set a background-image for the paragraph element:

Code:

<p style=" background-image: url(../images/background.jpg); padding: 30px; color: cyan; font-weight: bold; font-size: 50px; font-family: Arial, Helvetica, sans-serif; text-shadow: 5px -1px 0px blue; width: 90%; margin: auto;"> Hello I'm a paragraph with a background image. </p>

Outcome:

Hello I'm a paragraph with a background image.

By default, a background-image is placed at the top-left corner of an element, and is repeated both horizontally and vertically to cover the entire element.

The background-repeat Property

The background repeat property specifies how background images are repeated. A background image can be repeated along horizontal axis, vertical axis, or not repeated at all.

Code:

<div style=" width: 500px; height: 140px; background-image: url(../images/background.jpg); background-repeat: repeat-x; background-color: black; margin: auto;"> </div> <div style=" width: 500px; height: 300px; background-image: url(../images/background.jpg); background-repeat: repeat-y; background-color: blue; margin: auto;"> </div> <div style=" width: 500px; height: 50px; background-color: black; margin:auto;"> </div> <div style=" width: 500px; height: 300px; background-image: url(../images/background.jpg); background-repeat: repeat-xy; background-color: white; margin: auto;"> </div> <div style=" width: 500px; height: 50px; background-color: black; margin:auto;"> </div> <div style=" width: 500px; height: 100px; background-image: url(../images/background.jpg); background-repeat: no-repeat; background-color: blue; margin: auto;"> </div>

Outcome:

The background-size and background-position properties

The background-size, as you can probably guess is used to change the size of the image, background-position can be used to move the image.

Code:

<div style=" width: 500px; height: 500px; background-image: url(../images/background.jpg), url(../background2.jpg); background-repeat: no-repeat; background-size: 250px; background-position: 80px 350px, 80px 10px; background-color: black; margin: auto;"> </div>

Outcome:

The background-size property also takes the following keywords: contain and cover.

Contain keyword scales the image so that it fits the container, cover does the same, however, if the container and the image have a different aspect ratio the image will be cropped, pretty similar to how preserveAspectRatio="xMinYMin slice" attribute works in SVG.

The background-attachment Property

The background-attachment property sets wether a background image is fixed or scrolls with the rest of the page. Even if an element has scrolling mechanism, a fixed background doesn't move with the element.

The background-property has five values:

Value Description
scroll The background image wil scroll with the page
fixed The background image will not scroll with the page
local The background image will scroll with the element's contents
intial Sets property to it's default value
inherit Inherits it's property from parent element

The list-style-type Property

The list properties allow us to set different list item markers. In HTML there are three types of lists:

unordered list (<ul>) - the list items are marked with bullets
ordered lists (<ol>) - the list items are marked with number or letters
description lists (<dl>) - the list items can be assigned a description

With CSS lists can be styled further, and images can be used as the list item marker. One of the ways is to use the list-style-type property, which can be set to circle, square, decimal, disc, lower-alpha, etc.

HTML code:

<ol> <li id="circle"> Circle </li> <li id="square"> Square </li> <li id="decimal"> Decimal </li> <li id="disc"> Disc </li> <li id="lower-alpha"> Lower-alpha </li> </ol>

CSS code:

<style> #circle { list-style-type: circle; } #square { list-style-type: square; } #decimal { list-style-type: decimal; } #disc { list-style-type: disc; } #lower-alpha { list-style-type: lower-alpha; } ol { list-style-position: inside; margin-left: 45%; } </style>

Outcome:

  1. Circle
  2. Square
  3. Decimal
  4. Disc
  5. Lower-alpha

The list-style-image, list-style-position and content Properties

The list-style-image specifies an image to be used as the list item marker.
The list-style-position specifies the position of the marker box, it takes two values, inside and outside.
The content isn't a property only used for lists, it can probably be used for other things too, but I found it cool so I wanted to write about it, you can use it to add emojis as the list item marke, you can write li:before to make the emoji appear before the item, or li:after to make it appear after the item.

HTML code:

<ul id="onelist"> <li>Red</li> <li>Green</li> <li>Blue</li> </ul> <ul id="twolist"> <li>Red</li> <li>Green</li> <li>Blue</li> </ul>

CSS code:

<style> #onelist li { list-style-position: outside; border: 3px black solid; list-style-image: url(li.jpg); width: 370px; margin: auto; } #twolist li { list-style-position: inside; list-style-type: none; border: 3px black solid; width: 370px; margin: auto; } #twolist li:before { content: "❤"; padding-right: 5px; color: red; } </style>

Outcome:

  • Red
  • Green
  • Blue
  • Red
  • Green
  • Blue

The list-style Property

The list-style property is a shorthand property for setting list-style-type, list-style-image and list-style-position. It is used to set all of the list properties in one declaration:

Code:

<ul style="list-style: lower-alpha none inside; border: 3px black solid; width: 300px; padding-left: 0px; margin: auto;"> <li>Red</li> <li>Blue</li> <li>Green</li> </ul>

Outcome:

  • Red
  • Blue
  • Green

The table border Properties

The look of an HTML table can be greatly improved with CSS.

The border-collapse property specifies whether the table borders are collapsed into a single border or seperated as default. If the borders are seperated, the border-spacing property can be used to change the spacing.

HTML code:

<div class="bruh2"> <table> <tr> <td>Red</td> <td>Blue</td> </tr> <tr> <td>Yellow</td> <td>Red</td> </tr> </table> </div> <div class="bruh3" style="padding-top: 50px;"> <table> <tr> <td>Red</td> <td>Blue</td> </tr> <tr> <td>Yellow</td> <td>Red</td> </tr> </table> </div>

CSS code:

<style> .bruh2 table{ border: 3px solid black; margin: auto; width: 300px; height: 300px; border-collapse: separate; border-spacing: 10px 20px; background-image: url(../images/background.jpg); } .bruh2 td{ border: 3px solid black; background-color: cyan; } .bruh3 table{ border: 3px solid black; margin: auto; width: 300px; height: 300px; background-image: url(../images/background.jpg); border-collapse: collapse; } .bruh3 td{ border: 3px solid black; background-color: cyan; } </style>

Outcome:

Red Blue
Yellow Red
Red Blue
Yellow Red

The caption-side property specifies the position of a table caption. The values can be set as top or bottom.

The empty-cells Property

The empty-cells property specifies wheter or not to display borders and background on empty cells in table.

This property has two values, show value and hide value.

CSS code:

<style> #table-example td { border: 3px solid black; text-align: center; background-color: white; } #table-example { empty-cells: hide; } </style>

The table-layout Property

The table-layout specifies how the width of table columns is calculated.

The possible values are:

auto - when columns or cell width are not explocictly set, the column width will be in proportion to the amount of content in the cells that make up the column.

fixed - when column or cell width are not explicitly set, the column width will not be affected by the ammount of content in the cells that make up the column.

HTML code:

<p>Table-layout is set to <strong>auto</strong></p> <table class="auto"> <tr> <td width=“10%">500.000.000.000.000</td> <td width="90%">20.000</td> </tr> </table> <p>Table-layout is set to <strong>fixed</strong></p> <table class="fixed"> <tr> <td width="10%">500.000.000.000.000</td> <td width="90%">20.000</td> </tr> </table> </div>

CSS code:

<style> .bruh69 table { border-collapse: separate; width: 300px; border: 1px solid black; margin: auto; } .bruh69 td { border: 1px solid black; } table.auto { table-layout: auto; } table.fixed { table-layout: fixed; } </style>

Outcome:

Table-layout is set to auto

500.000.000.000.000 20.000

Table-layout is set to fixed

500.000.000.000.000 20.000

Setting styles to links

Links can be styled with any CSS property (e.g., color, font-family, background, etc.). In addition, links can be styled differently, depending on what state they are in. The following pseudo selectors are available

a:link - defines the style for normal unvisited links
a:hover - defines the style for link when mouse is hovered over it
a:active - defines the style for link when you click on it
a:visited - defines the style for link after you have already visited the url

HTML code:

<a id="linkexample" href="https://www.sololearn.com/Courses/" target="_blank"> Click me please I'm a link and I'm very very sad... </a>

CSS code:

<style> #somethingbruh a:link{ color: blue; text-decoration: underline } #somethingbruh a:visited { color: green; } #somethingbruh a:active { color: red; } #somethingbruh a:hover{ color: red; } </style>

Outcome:

Click me please I'm a link and I'm very very sad...

Remember, if we had put a:hover first for example it wouldn't work, because it would be overran by the other rules, the rules should be written in the correct order, a:active and a:hover should be last so they work at all times, a:visited should be second after a:link.

Setting the Mouse Cursor Style

CSS allows you to set your desired cursor style when you mouse over an element. For example, you can change your cursor into a hand icon, help icon, and much more, rather than using the default pointer.

Code:

<p style="cursor: help;"> Do you need help my child? maybe some food huh? </p>

Outcome:

Do you need help my child? maybe some food huh?
