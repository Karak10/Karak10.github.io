About the position property

There are four different position values you can use, static which is the default one, relative, absolute, fixed and sticky.

The static position is the position we are all more familiar with, since it's the defaul position, every HTML element gets positioned one under another and no element can be place one over the other.

The relative position works pretty much like the static position, with one exception, you can use the top, right, bottom and left properties to move the element relatively to it's original position.

Lets make a fake table for example to use for our examples:

HTML Code:

<div class="parent"> <div class="row"> <div class="title"> Parent </div> </div> <div class="row"> <div class="one"> One </div> </div> <div class="row"> <div class="two"> Two </div> </div> <div class="row"> <div class="three"> Three </div> </div> </div>

CSS Code:

<Style> .parent { display: table; background-color: rgb(0, 162, 255); font-weight: bold; font-size: 30px; border-collapse: collapse; margin-left: auto; margin-right: auto; } .row { display: table-row; } .title { display: inline-block; background-color: rgb(0, 162, 255); border: 10px solid rgb(0, 162, 255); border-bottom: 0px solid blue; height: 50px; width: 580px; } .one { vertical-align: middle; padding-left: 10px; display: inline-block; background-color: rgb(155, 221, 129); border: 10px solid rgb(0, 162, 255); border-top: 0px solid blue; height: 50px; border-bottom: 0px solid blue; width: 580px; } .two { vertical-align: middle; padding-left: 10px; display: inline-block; background-color: rgb(245, 91, 91); border: 10px solid rgb(0, 162, 255); border-top: 0px solid blue; height: 50px; border-bottom: 0px solid blue; width: 580px; } .three { vertical-align: middle; padding-left: 10px; display: inline-block; background-color: pink; border: 10px solid rgb(0, 162, 255); border-top: 0px solid blue; height: 50px; width: 580px; } </Style>

The absolute position works like relative, however, the original position of the element is not rendered at all, and you will have to position the element relatively to the start of the page, since it doesn't have an original position.

The page however may be big, and it also can change, therefore, we can also use a trick to position an absolute element relatively to it's parent, simple by making it's parent relative.

<div class="parent" style="position: relative;">...
...<div class="one" style="position: absolute; top: 40px;">

As you can see the second and third cells have filled the empty space of the first cell, just like if the first cell wasn't there.

The fixed position works like absolute, meaning that you position the element relatively to the start of the page, the difference is that when an element is fixed it will stay at the same position as you scroll, useful if you need a navigation bar or something.

The sticky position works like realative and fixed together, meaning, the element will be positioned relatively to it's original position, and it will also follow your screen while scrolling, if you for example write top: 0px it will start following the screen while scrolling only after you have scrolled over it and it has met the start of the page, since we wrote top: 0px, it will then only stop following the screen after it meets it's original position again.

You need to remember one thing about sticky position, it will only stick on the page and follow the screen till it reaches the end of it's parent, if the parent is the body then it will not have an end.

Also, only 50% of the browsers support the sticky position, to increase that number to 90% you can simply just add the follow positions which work as alternatives if a browser doesn't support position: sticky:

position: -webkit-sticky;
position: -moz-sticky;
position: -ms-sticky;
position: -o- sticky;
position: sticky;

If you want to have an alternative behavior in case a browser doesn't support any of these positions you can write the following code:

<style> .example { position: relative/absolute/static or whatever you want; } @supports (position: sticky) { .example { position: -webkit-sticky; position: -moz-sticky; position: -ms-sticky; position: -o- sticky; position: sticky; top:0; } } <style>

This will make the element have a specific position unless position: sticky; is supported, in which case the position will be sticky.

The float property

With float property, we can position elements one afer another, or, even allow inline elements such as text wrap around a specific element, such as an image for example.

The values for the float property are left, right and none.

Left and right float elements in those directions, respectively, none (the default) ensures that the element will not float.

Below is an example of an image that is floated to the right.

Code:

<p style=" width: 450px; margin: auto; background-color: cyan; border: 3px solid black; border-radius: 10px; padding: 0px 0px 10px 10px; text-align: justify;"> <img src="../images/background.jpg" style=" float: right; margin-left: 10px;"> This paragraph has an image that is floated to the right. It is highly recommended to add a margin to images so that the text does not get too close to the image. If you want your text to be easily read, you should always add a few pixels between words and borders, images, and other content. </p>

Outcome:

This paragraph has an image that is floated to the right. It is highly recommended to add a margin to images so that the text does not get too close to the image. If you want your text to be easily read, you should always add a few pixels between words and borders, images, and other content.

As you can see the text has wrap around the image.

The clear property

So, normally, as it's explained in the video above, when an element floats, the elements below it moves up under it to fill the empty space, it's explained in the video that the clear property can be used to set some kind of invisible wall over the elements, which wall hits with the floating elements no allowing the element to move under the floating elements.

The clear property works together with float property, and it takes three values, right, left and both. If you set the invisible wall at the left side of an element no floating element can be at it's left, same goes for the right values, but the opposite direction, usually you just use the both value, which means that the element cannot move under any floating elements.

The overflow Property

There are three overflow properties, the overflow-x, the overflow-y and the overflow properties.

So, some times, the content of an element is bigger than the size of the element, as a result, the content goes beyond the borders of it's element, see this paragraph for example:

Code:

<p style=" width: 200px; border: 3px solid black; margin: auto; background-color: white;"> AAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAA </p>

Outcome:

AAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAA

To fix that, we can use the overflow-x property, there are actually four value the overflow properties can get, visible which is the dafault, meaning that the content outside the border is visible, scroll which hides the content that's outside the border and adds a scroll bar that allows us to scroll, hidden that just hides the content that surpasses the border, and auto, which is useful when you use percentage to set the width or height of an element, since for some screens the content will fit and for some it won't, using the auto value will make a scroll bar appear only when it's needed.

Code:

<p style=" width: 200px; border: 3px solid black; margin: auto; background-color: white; overflow-x: scroll;"> AAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAA </p>

Outcome:

AAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAA

The overflow-y property works the same way, its just used for the y axis, the overflow property is for both.

The z-index property

The z-index property is used to create layers, for example, normally, an element added after another in HTML will be infront if moved by position property, if we wanted to change that we could set layers, an element with z-index: 1 is infront an element with z-index: 0 and behind an element with z-index: 2.
