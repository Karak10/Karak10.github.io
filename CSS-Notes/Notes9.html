The display Property

Every element on a web page is a rectangular box. The display property determines how that rectangular box behaves.

A block element is an element that takes up the fullest width available, it's width and height can be changed and you can't place two block elements one next ot each other, since a block element forces a line break before and after it.

An inline element on the other hand can be placed inside a block element, and you can place many inline elements one after another, their width and height however can't be changed.

An inline-block element behaves like an image, you can place it inside a block element, you can place many of these one after another, but you can also change it's width and height.

HTML code:

<p class="inline-element size"> Helli I'm an inline paragraph</p> <p class="inline-element size"> Helli I'm an inline paragraph</p> <p class="block-element size"> Helli I'm a block paragraph</p> <p class="block-element size"> Helli I'm a block paragraph</p> <p class="inline-block-element size"> Helli I'm an inline-block paragraph</p> <p class="inline-block-element size"> Helli I'm an inline-block paragraph</p>

CSS code:

<style> .size { width: 100px; height: 60px; background-color: red; border: 3px solid black; margin: auto; } .inline-element { display: inline; } .block-element { display: block; } .inline-block-element { display: inline-block; } </style>

The align-items property

The align-items property is used to position the items on the cross axis, the default value is stretch, which makes the items stretch to fill the whole cross axis, thats why the items even tho they had different height when put in the flex box took the same height, because we used min-height, we haven't specified the max-height, so they can grow as much as they need, for width however we used width, therefore they remain at this specific width and don't fill the cross axis when flex-direction: column;, I hope you understand.

This property uses pretty much the same values that justify-content uses, it just spaces them on the cross axis.

align-content and flex-wrap properties

The align-content property can be visible on multi lined flexboxes, the align-item will position the items on their line, the align-content will position all the items to be at start, center, enter, etc, it takes same values with justify-content, and positions the items on the cross axis.

The flex-wrap: wrap; property can wrap the items, normally, when we changed the size of the browser the items would shrink, with flex-wrap: wrap; however they get one under another, lets try this:

<div class="flex-container" style="justify-content: flex-start; flex-wrap: wrap; align-items: flex-start; align-content: flex-end; height: 600px;">

flex-shrink, flex-grow and flex-basis properties

When the size of the browser changes, the items shrink by default, you can change the way they shrink with the flex-shrink property, this property should be used on the items and not on the container, you could write for example flex-shrink: 0; and the item wouldn't shrink at all, flex-shrink: 1 and the item would shrink, flex-shrink: 2 and the item would shrink twice as much as the one with flex-shrink: 1.

The flex-grow works the same way, it defines how the items will grow when the size of the browser gets bigger, but you need to remember, if for example we had two items and used flex-grow: 1; on first and flex-grow: 2; on second, the second item wouldn't be twice the size of the first, because it would just take twice of the empty space on the browser, and since we have already defined that the start width of the items is 200px, the second item wouldn't be exactly twice of the first, lets say the empty space was 450px, first item would take 150px and second would take 300px, so first item would be 350px and second would be 500px since their start width is 200px, as you can see the second item isn't twice the size of the first.

To fix that, you can use the flex-basis property, if you write flex-basis: 0; on both items the start width will be ignored, and the second item will always be twice as big to the first item.

align-self and order properties

The align-self property works exactly like align-items property, but it can be used in each item seperately, to align items differently, it uses same values with the align-items property.

The order property can be used to change the order of the items without changing the HTML, simply by writing order: 1;, order: 2;, etc, but it isn't recommended to use it, because screen readers don't still read the HTML order, and there are also other problems.

The flex property

The flex property is a shortcut for flex-shrink, flex-grow and flex-basis properties, it uses three paremeters, firt is used for the flex-grow, second for the flex-shrink and third for the flex-basis.

About display: grid;

Warning: when I made this section I was still a beginner and the explanation of grid layout you will see bellow is not very well written, I remade this section in my responsive web design notes where I explain grid way better.

The display: grid; is pretty similar to display: flex;, the difference is that while the display: flex; makes the content flexible in only one direction, the display: grid; makes the content flexible in all directions.

Just like when we create a flexbox, we first need to make our grid-container

HTML code:

<div class="grid-container"> <div class="grid-item grid-item-1" > Item 1</div> <div class="grid-item grid-item-2" style="background-color: rgb(0, 174, 255);">Item 2</div> <div class="grid-item grid-item-3">Item 3</div> </div>

CSS code:

<style> .grid-container { display: grid; background-color: gray; border: 3px solid black; } .grid-item { background-color: cyan; border: solid white; min-width: 100px; min-height: 100px; font-size: 40px; word-spacing: 390px; } </style>

grid-auto-columns and grid-auto-rows properties

You can use these properties to define what the default width/height of columns/rows should be if you haven't defined it already, thats very useful when you are using many items but only want to size few.

Something I want to add, which can be used for both grid-auto-rows/columns and grid-template-rows/columns is the minmax() value, before I spoke about the reapeat(), the minmax() value is another helpful keyword used to define the minimum and maximum width/height a column/row can get based on the content it has inside it, if for example I wanted to make it so the more words that get written inside the item the taller it gets, I would write grid-auto-rows: minmax(150px, auto);, this would make all rows that aren't defined have 150px height, unless the content inside of them doesn't fit, in which case they get taller, pretty cool huh?

The justify-content, align-content, justify-items, align-items, align-self and justify-self properties

Some of these properties are also used on flexbox, so there isn't really much to speak about these, the only new property we see is the justify-items property, which works pretty much the same with align-items, but for the x axis, this property doesn't exist in flexbox because it isn't needed.

Something important you need to remember is the when you want to align the content or items to the start or end you don't write flex-start/end like when we used flexbox, you simply write start/end.

The justify-items property by default has the stretch value, which makes the items stretch to fill the screen on the x axis, this is only seen if you haven't specified a width or max-width, this property can take many values, such as end, start, center, stretch, etc.

The justify-content property aligns the content, by default it's value is start, it can like I said in the flex-box section take few values such as start, center, end, space-between, space-around, etc.

The align-items and align-content work the same as the justify properties, the only difference is that they align the content/items vertically on the y axis.

The align-self and justify-self properties work exactly like align-items and justify-items but you can use the properties on each item seperately, these properties override the justify-items and align-items properties.
