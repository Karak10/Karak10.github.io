Introduction to CSS

What is CSS?

CSS stands for Cascading Style Sheets.

Cascading refers to the way CSS applies one style on top of another.
Style Sheets control the look and feel of web documents.

CSS and HTML work hand in hand:

HTML sorts out the page structure.
CSS defines how HTML elements are displayed.

Why use CSS?

CSS allows you to apply specific styles to specific HTML elements.

The main benefit of CSS is that it allows you to seperate style from content.

Using just HTML, all the styles and formatting are in the same place, which becomes rather difficult to maintain as the page grows.

Inline, Embedded, External CSS

Inline CSS:

Using an inline style is one of the ways to insert a style sheet. With an inline style, an unique style is applied to a single element.

In order to use an inline style, add the style attribute to the relevant tag.

The example bellow shows how to create a paragraph with a gray background and white text:

Code:

<p style="color:white; background-color:gray;"> This is an example of inline styling. <p>

Outcome:

This is an example of inline styling.

Embedded/Internal CSS:

Internal styles are defined with the <style> element, inside the <head> section of an HTML page.

For example, the following code styles all paragraphs:

<head>
<style>
p {
color:white;
background-color:gray;
}
</style>
<head>

External CSS:

With this method, all styling rules are contained in a single text file, which is saved with the .css extension.

This CSS file is then referenced in the HTML using the <link> tag. The <link> element goes inside the <head> section.

Here is an example:

<head>
<link rel="stylesheet" href="example.css">
<head>

Rel stands for relationship, so you specify whats the relationship with the file, in which case its a stylesheet.

The other file uses the same type of code I described before in the embedded/internal section, the only difference is that instead of typing it inside the style tag in head we type it in another file.

Something else I want to add is that when you have updeaded an external CSS file to your server in an online website the cache saves your CSS file and when you update your file it still uses the old one, to force it read the new one you can write name.css?version=1, name.css?version=2 etc, you only write it in code, you don't need to change the name of your file, you could actually write anything, not just version, this lets the browser know the file has changed, so it looks for the CSS file and updates it.

The External CSS style can be overran by an Embedded/Internal CSS style, which can be overran by an inline CSS style, they all can be overran by the !important attribute.

CSS Syntax

CSS is composed of style rules that the browser interprets and then applies to the corresponding elements in your document.

A style rule has three parts: selector, property and value.

For example, the headline color can be defined as:

h1 { color: orange; }

The most common and easy to understand selectors are the type selectors. This selector targets element types on the page.

For example, to target all the paragraphs on the page:

p {
color: red;
font-size: 130%;
}

A styling rule written after another can overrun the previous rule, any styling rule in which you write !important can overrun another styling rule no matter it's position.

Class and id Selectors

id selectors allow you to style an HTML element that has an id attribute, regardless of their position in the document tree. Here is an example of an id selector:

HTML code:

<p id="intro"> This paragraph is in the intro section. </p>
<p> This paragraph is not in the intro section. </p>

CSS code:

#intro {
color: white;
background-color: gray;
}

Outcome:

This paragraph is in the intro section.

This paragraph is not in the intro section.

Class selectors work in a similar way. The major difference is that IDs can only be applied once, while classes can be used as many times on page as needed. Why use ids then? Well, id is mostly used to reference an element while using JavaScript, ids are also more important than classes, therefore, an id will overrun a class, let me give you an example:

HTML code:

<p class="white"> This is a paragraph. </p>
<p class="white" id="blue"> This is a paragraph. </p>

CSS code:

#blue {
color: blue;
}
.white {
color: white;
background-color: gray;
}

Outcome:

This is a paragraph.

This is a paragraph.

As you can see, to reference an id I write #name, to reference a class I write .name.

You could do the same thing using multiple classes, to use multiple classes you simply make a space and write the name of your second class, lets try this:

HTML code:

<p class="white"> This is a paragraph. </p>
<p class="white blue"> This is a paragraph. </p>

CSS code:

.white {
color: white;
background-color: gray;
}
.blue {
color: blue;
}

Outcome:

This is a paragraph.

This is a paragraph.

It only works because the styling rule for blue class was written after the styling rule for white class, if we had done the opposite the blue class would not overrun the styling rules of white class, while id would regardless it's position in the document tree, thats because id is more important than class.

HTML code:

<p class="white"> This is a paragraph. </p>
<p class="white blue"> This is a paragraph. </p>

CSS code:

.blue {
color: blue;
}
.white {
color: white;
background-color: gray;
}

Outcome:

This is a paragraph.

This is a paragraph.

These styling rules, when used on a bigger extend can be confusing, thats why you can write !important inside the styling rule to force it overrun any other styling rules.

HTML code:

<p class="white"> This is a paragraph. </p>
<p class="white blue"> This is a paragraph. </p>

CSS code:

.blue {
color: blue !imporant;
}
.white {
color: white;
background-color: gray;
}

Outcome:

This is a paragraph.

This is a paragraph.

Descendant Selectors

These selectors are used to select elements that are descendants of another element. When selecting levels, you can select as many levels deep as you need to.

For example, see the code below, I will target only the <em> element inside the second paragraph inside the <div>:

HTML code:

<div id="example1">

<p> This is a <em>paragraph</em>.</p>

<p class="secondp">This is a <em>second paragraph</em>.</p>

</div>

<p class="secondp"> This is a <em>third paragraph</em>.</p>

CSS code:

#example1 .secondp em {
color: pink;
}
.secondp {
background-color: gray;
color: white;
}

Outcome:

This is a paragraph.

This is a second paragraph.

This is a third paragraph.

How to make CSS comments

Comments are used to explain your code, and my help you when you edit the code later, if you work with a team comments should always be written, so your team can read your code and understand it easier and faster.

A CSS comment looks like this:

/* Comment goes here */

The comments do not appear in the browser, they are readable however through the code, and the code can be easily be seen by visitors, so don't include important informations in the comments, simply use these to describe what each part of the code is supposed to does.

Style Cascade and Inheritance

Cascade:

The final appearance of a web page is a result of different styling rules.

The three main sources of style information that form a cascade are:

-The stylesheet created by the author of the page.

-The browser's default styles

-And styles specified by the user

Inheritance:

Inheritance refers to the way properties flow through the page. A child element will usually take the characteristics of the parent element unless otherwise defined.

See the following example to understand what I mean:

Code:

<head>
<meta charset="UTF-8">
<title> Example </title>
<style>
body {
color: green;
font-family: Arial;
}
</style>
</head>

With this code, if we now added a paragraph without specifying what color or font-family we want it to be, it would get the characteristics of it's parent lement, and it would be green with Arial font-family.
